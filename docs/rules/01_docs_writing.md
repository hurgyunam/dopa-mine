# 문서 작성/운영 규칙

이 문서는 `docs` 하위 문서의 작성 방식과 동기화 기준을 정의한다.

## 1. 기본 구조
- `docs` 루트에는 `README.md`, `TODO.md`만 둔다.
- 상세 문서는 목적별 하위 폴더(`context`, `product`, `design`, `architecture`, `todo`, `issues`)에 배치한다.
- 각 문서는 단일 H1을 사용하고 주요 섹션은 `##` 기준으로 정리한다.

## 2. 문서 작성 원칙
- 용어는 `docs/product/01_glossary.md` 기준을 우선 적용한다.
- 긴 설명보다 결정 근거와 운영 기준을 우선 기록한다.
- 문서 간 충돌 시 `docs/README.md`의 우선순위 정책을 따른다.

## 3. 동기화 원칙
- 기능/정책 변경 시 관련 문서를 최소 2개 이상 동기화한다.
- 코드 변경 작업은 문서 갱신 확인 없이 완료로 간주하지 않는다.
- 라우트 변경이 있으면 `docs/architecture/05_navigation_routes.md`를 같은 작업에서 갱신한다.

## 4. 변경 이력
- `docs` 핵심 문서는 하단 `## 변경 이력` 섹션에 표 형식으로 기록한다.
- 기본 컬럼: `날짜(YYYY-MM-DD) | 변경 요약 | 작성자`
