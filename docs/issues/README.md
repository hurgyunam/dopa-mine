# Issues Logging Guide

목적은 하나다:  
`AI가 준 제안`과 `내가 한 판단`을 분리해 남겨, 나중에 봐도 출처가 투명하게 보이게 한다.

## 규칙 (간단)
- 채팅 1개 = 파일 1개
- 파일명: `CHAT_YYYY-MM-DD_<topic>.md`
- 통합 인덱스: `ISSUES.md` (링크만 관리)
- 각 `CHAT_*.md` 마지막에 `## 취합용 요약` 섹션을 반드시 추가
- `취합용 요약`은 3줄 고정: `결론`, `AI 제안 핵심`, `사용자 결정 핵심`
- 각 줄은 80~120자 내로 간결하게 작성

## 채팅 문서 최소 양식
```md
# Chat Log - YYYY-MM-DD - <topic>

## 왜 기록하나
- AI 제안과 사용자 결정을 구분해 남긴다.

## 결정 요약
- 결론:
- 이유:

## 제안 출처
### AI 제안
- <무엇을 제안했는가>

### 사용자 제안
- <무엇을 요구/결정했는가>

## 반영
- 변경 문서: `docs/...`
- 상태: done | pending

## 취합용 요약
- 결론:
- AI 제안 핵심:
- 사용자 결정 핵심:
```
